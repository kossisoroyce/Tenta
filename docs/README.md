## Documentation Categories

- `architecture/`: system components, runtime topology, storage, policy, and schema.
- `design/`: product and interface design decisions.
- `algorithms/`: drift detection, online learning, healing logic, and evaluation methods.
- `api/`: service contracts, endpoints, event payloads, and SDK surfaces.
- `deployment/`: Kubernetes, security, testing, release, and operations notes.
- `ui/`: dashboard views and operator workflows.
- `sdk/`: plugin and extension contracts.
- `research/`: experiments, papers, open questions, and longer-term roadmap.
- `user-guide/`: user-facing onboarding and usage guidance.
